Qué se probó

El broker (Node 24, sin dependencias nativas) escucha en 127.0.0.1:8890, verifica el secreto por hook, persiste el evento en SQLite y despacha al adaptador spawn:claude. Ahí vive la regla que pediste:

POST /hook/e2e-real header sessionId presente? │ ├─ no → spawn: claude -p "<prompt>" --output-format json │ └─ sí → spawn: claude --resume <sessionId> -p "<prompt>" --output-format json

Se registró un hook real (e2e-real, modo trigger) con workdir apuntando al repo free-code (donde vive la sesión d63be319), y un prompt que le pide explícitamente al agente no tocar archivos ni usar herramientas — solo confirmar recepción.

Bug encontrado y corregido durante la prueba

fix

adapters/spawn-runner/claude.ts construía claude --resume <id> "<prompt>" sin -p. Sin ese flag, --resume abre el picker interactivo — en un proceso hijo sin TTY eso se hubiera colgado indefinidamente en cuanto un hook real disparara la rama de resume.

Corregido a claude --resume <id> -p "<prompt>" --output-format json antes de correr el Test B. Confirmado en el log real: terminó en 6.4s con exit 0.
